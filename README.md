#Заданее 1 Резервное копирование


1.1 Восстановление данных в полном объёме за предыдущий день

Сделал бы полный бэкап  раз в сутки ,тогда если что-то сломалось просто восстанавливаем БД из вчерашней полной копии.

1.2 Восстановление данных за час до предполагаемой поломки

Тут нужен вариант, чтобы можно было откатиться к нужному времени, а не только к моменту бэкапа.

Для этого подходит схема,базовая  копия + журнал изменений.

В PostgreSQL это WAL, и с ним можно сделать восстановление Point-in-Time Recovery (восстановление к моменту времени), например на состояние “за час до аварии”.

#Задание 2 PostgreSQL

2.1 Пример резервного копирования и восстановления

Резервная копия  через pg_dump

pg_dump -U postgres -d mydb -F c -f mydb.dump

Восстановление через pg_restore


pg_restore -U postgres -d mydb mydb.dump

#Задание 3 MySQL

3.1 Пример инкрементного резервного копирования

В MySQL в качестве инкремента можно использовать binlog это изменения, которые происходили после полного бэкапа.

Пример выгрузки событий из binlog

mysqlbinlog /var/lib/mysql/mysql-bin.000001 > inc_000001.sql

Потом при восстановлении сначала поднимаем полный бэкап, а затем применяем изменения из binlog до нужного момента.
